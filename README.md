Лабораторна робота №3
"Симуляція алгоритмів планування процесів"
Дисципліна: Системне програмування

Мета:
Продовжити розробку симуляції алгоритмів планування процесів, розпочату в попередній лабораторній роботі, шляхом додавання нових алгоритмів та функціональності. 
Дослідити вплив різних алгоритмів на ефективність роботи системи та проаналізувати їх переваги та недоліки.

1. Опис та структура симуляції

У програмі моделюється робота процесорного планувальника.
Кожен процес має параметри:
- ID
- Час прибуття (arrival)
- Час виконання (burst)
- Пріоритет

2. Реалізовані алгоритми
2.1 FCFS (First-Come-First-Served)
- Планування у порядку прибуття.
- Простий, але може формувати черги та великий час очікування.

2.2 Round Robin

- Кожен процес отримує фіксований квант часу.
- Підходить для багатозадачних систем та ОС.
- Реалізовано за допомогою циклічної черги.

2.3 Priority Scheduling (non-preemptive)
- Процеси виконуються за пріоритетами.
- Недолік: можливе голодування низькопріоритетних процесів.

2.4 Dynamic Priority Scheduling (preemptive, with aging)
- Процес, що довго чекає → отримує підвищення пріоритету.
- Виконується preemptive-перемикання кожну одиницю часу.
- Реалізує боротьбу з голодуванням.

2.5 SJF (Shortest Job First)
- Процес із найменшим burstTime виконується першим.
- Непереривний варіант (non-preemptive).
- Один із найефективніших алгоритмів щодо середнього часу очікування.

Висновки
На основі проведеного аналізу:
- SJF показує найнижчий середній час очікування — підтверджує теоретичну оптимальність SJF для середнього часу очікування.
- Priority Scheduling може призводити до голодування процесів з низькими пріоритетами.
- Dynamic Priority + Aging повністю вирішує проблему голодування та забезпечує стабільний розподіл CPU.
- Round Robin добре підходить для систем із великою кількістю коротких задач, але збільшує середній час очікування.
- FCFS — найпростіший, але найменш ефективний алгоритм у змішаних навантаженнях.
